# AI Experience Pattern References

This folder should contain visual examples (screenshots/images) from the design deck showing do's and don'ts for AI UX patterns.

## Suggested naming convention

Use descriptive names that indicate the pattern type and whether it's a positive or negative example:

- `loading-states-do.png` - Good examples of AI loading states
- `loading-states-dont.png` - Anti-patterns to avoid
- `input-patterns-do.png` - Good AI input design
- `input-patterns-dont.png` - Bad AI input design
- `output-display-do.png` - Good ways to show AI output
- `output-display-dont.png` - Poor output presentation
- `error-handling-do.png` - Good AI error states
- `error-handling-dont.png` - Bad error handling
- `transparency-do.png` - Good transparency/control patterns
- `transparency-dont.png` - Poor transparency
- `streaming-do.png` - Good streaming UI
- `streaming-dont.png` - Bad streaming patterns

## How to populate

1. Export slides from the deck as images (File → Download → PNG)
2. Rename them using the convention above
3. Place them in this folder
4. The skill will read and display relevant images when building AI experiences
